long game

June 2019 - September 2019

ONBOARDING TOUR

 

Original tour

Through the observations of previous user testing, we concluded that the onboarding tour wasn’t engaging the user and effectively communicating the core mechanisms of the game or persuasively convincing the user to sign up. Therefore, I was given the task to redesign the onboarding tour, impacting the user experience for thousands of users.

General aims were to increase: use of brand elements, user interest/engagement, general understanding of app purpose & functions, and incorporate a fun, playful style.

I incorporated updated, on-brand graphics and worked closely with Long Game’s COM, Senior Product Designer, and Head Game Designer to reiterate initial designs.

I then proceeded to conduct four rounds of user testing, adjusting order of slides, text, and artwork accordingly.

Finally, I presented insights and findings to the rest of the company.

 

Presentation after first round of user testing outlining findings & potential solutions that could be further developed

(Designed by Long Game’s Senior Product Designer Liz Hewell-Garrett)

Issues identified through user testing:

  • Users would rarely read all of the text and would quickly skip through the slides

  • Due to their lack of information, users would become confused later in the onboarding process (e.g. what are Coins for and why am I being rewarded them for signing up?)

  • Did not fully communicate to users what exactly Long Game was (when prompted to describe their biggest issues with the app, almost half of users expressed confusion as to how it actually works)

Potential solutions and next steps:

  • Reduce text and create clearer graphics to promote legibility and efficiently communicate essential information

    • This would also encourage users to more fully digest the slides

  • Prioritize what is necessary and avoid bombarding users with too much new information

  • Rethink order and copy to create a more succinct and easily understandable narrative for what Long Game is


ROUGH BRAINSTORM

Attempted fixes:

  • Added gradient overlay to reduce background noise and emphasize information presented

  • Greatly reduced text (copy written by myself and Head Game Designer as sample)

  • Created legible graphic explaining the core loop and how Long Game works

    • Also visually introduced crucial concepts such as “Coins” and “Jars” later used in the onboarding process

  • Reordered the tour to revolve around the core loop, building a more cohesive narrative structure

Issues (not tested, general feedback from Long Game’s Senior Product Designer and Head Game Designer):

  • Background art not very connected/representative of in-game art

  • The UI graphic on the “Deposit” screen wasn’t as clear as possible—small text and unfamiliar UI could be confusing for users

  • Gradient overlay not exactly on-brand—although text and graphics are clearer, still want to maintain/preserve playfulness and vividness of artwork

 

Attempted fixes:

  • Updated copy according to COM’s adjustments

  • Replaced the graphics in the “Save in Your Jars” screen (previously “Deposit”) to be more visually interesting and engaging

  • Replaced background art with in-game maps (at the time, I used a set of assets that hadn’t yet been knitted together)

    • Opted for a vertical scroll format for a more playful/less standard experience

Issues (feedback from Long Game as well as user testing):

  • Needed more emphasis on the check card (a new feature Long Game was going to implement in September)

  • “Earn Coins” screen didn’t seem to be very helpful (did not add to the user’s impression of Long Game in a meaningful way)

  • “Save in Your Jars” screen also had a similar issue—additionally, it was a little unclear as to what “Jars” were

  • Button on last screen should be adjusted to be more playful to make the app feel more like a game


MY FINAL DESIGN

Attempted fixes (after three rounds of user testing):

  • Updated to correct map assets

  • Added screen for check card

    • Optimal placement and copy determined by user testing—Long Game wants to prioritize this feature

  • Updated graphic for core loop to add numbering and clarify the process

  • Eliminated “Earn Coins” screen

  • Designed new graphics for previous “Save in Your Jars” screen to visually communicate purpose and function of Jars, thus allowing the removal of “Jars” in the title

  • Animated prototype (as seen in the Marvel prototype above) such that clouds, text, then button fade in, allowing appreciation of artwork and prevent users from simply skipping through all given information

  • Narrative is now more feature-focused, and concisely takes the user through the different mechanisms of the app

Issues I’ve identified:

  • Needs to undergo more user testing

  • Card screen seems a bit bare and unconnected—how does it fit into Long Game’s core loop and functions? What are some benefits of using the card?
